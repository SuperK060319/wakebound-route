# 《沙海机城》第一版规划

## 1. 产品契约

《沙海机城》是一款原创浏览器牌组构筑 Roguelike。玩家操纵一座穿越沙海的移动机城，通过“指令牌组 + 城芯轨环”双构筑，应对拦截机械、沙暴异兽与失控的古代巨构。

- 平台：电脑浏览器、手机横屏。
- 输入：点击卡牌，再点击合法目标；不依赖拖拽、右键或悬停。
- 单局时长：垂直切片约 15–25 分钟。
- 完整循环：标题 → 路线图 → 战斗 → 奖励 → 路线选择 → 首领 → 结算。
- 公开交付：静态网页部署，打开链接即可玩，无需登录和安装。
- 原创边界：角色、世界观、卡名、文本、视觉、音效、数值和代码全部原创；只借鉴牌组构筑、敌人意图、路线选择等类型结构。

第一版不做永久天赋树、长篇剧情、每日挑战、多人功能、账号系统和大量角色。

## 2. 第一版可玩垂直切片

### 内容范围

- 1 座机城：“曙脊号”。
- 1 套起始牌组：10 张牌。
- 18 种可获得指令牌，分为推进、防护、火力、调度四类。
- 6 种城芯模块，每局最多安装 3 个。
- 2 种普通敌人、1 组双目标战斗、1 个两阶段首领。
- 1 张由固定种子生成的四层路线图。
- 3 类奖励：选牌、选模块、修复机城。
- 本地自动存档、继续游戏、重新开始、音量和文字速度设置。

### 路线结构

```text
第1层：两种普通战斗二选一
          ↓
第2层：模块工坊 / 修复站二选一
          ↓
第3层：单体重甲敌人 / 双目标伏击二选一
          ↓
第4层：首领“埋日构装体”
```

路线节点与连线由同一份 `routeLinks` 数据生成。只有与上一个节点真实相连的节点才能点击，不能出现“画面分叉、实际路线固定”的假选择。

### 战斗基础

- 每回合获得 3 点“调度值”，抽至 5 张手牌。
- 攻击造成结构伤害；防御生成当回合有效的装甲。
- 敌人提前显示下一行动：攻击、蓄势、防护、召唤或干扰轨环。
- 玩家点击牌后进入目标选择：
  - 攻击牌高亮合法敌人；
  - 防护和调度牌高亮机城或轨环；
  - 再点原牌、空白区域或按 Escape 可取消；
  - 点击无效目标不扣资源、不弃牌。
- 多敌人战斗用于验证目标选择没有被单目标原型掩盖。
- “结束调度”按钮始终可见，并显示敌方即将造成的总伤害。

## 3. 原创双构筑：指令牌组 × 城芯轨环

这不是“牌组加一排被动饰品”。第二构筑轴会直接改变玩家每一张牌的出牌顺序和奖励选择。

### 第一轴：指令牌组

每张牌除费用、目标和效果外，还有一个“行程值”：

- 短令：轨芯前进 1 格。
- 长令：轨芯前进 2 格。
- 定标令：轨芯停留。
- 逆转令：轨芯反向移动。

牌组构筑不仅考虑伤害或防御，还要考虑行程值分布。

### 第二轴：城芯轨环

战场上有一个由四个站位组成的可视轨环，轨芯指针在其中移动。最多安装三个模块，留一个空位。每打出一张牌，结算其效果后，轨芯按该牌的行程值移动；落点模块立即触发。

第一版模块示例：

| 模块 | 触发规则 | 造成的决策 |
|---|---|---|
| 磁轨投射台 | 轨芯移动恰好两格后落入，向刚才选中的敌人追加伤害 | 长令牌获得火力价值，玩家会保留合适牌序 |
| 承压拱架 | 防护牌结算后落入，将部分装甲保留到下一回合 | 玩家可能延后防御牌，只为踩准站位 |
| 砂晶棱镜 | 火力牌落入后，下一张单体攻击改为相邻扩散 | 双敌战中改变目标和出牌先后 |
| 分令枢纽 | 落入后，下一张牌行程值减 1，最低为 0 | 可连续触发同一模块，但会牺牲轨环推进 |
| 蓄势飞轮 | 每次经过而非落入时积累一层；三层后下一张牌费用减 1 | 鼓励长令和持续绕环，而非反复停留 |
| 逆相闸门 | 逆向进入时复制下一次模块触发，随后进入休眠 | 让逆转牌成为构筑核心，而非普通位移牌 |

### 为什么它真的改变决策

玩家每回合必须同时回答三个问题：

1. 这张牌现在的直接效果是否值得？
2. 它会让轨芯停在哪里？
3. 为了触发关键模块，是否应该改变牌序、目标或保留某张牌？

例如，敌人即将造成 12 点伤害。玩家手里有一张高防御牌，但轨芯距“承压拱架”两格。立即打防御可以安全度过本回合；先打一张低收益短令再防御，则能保留装甲到下一回合，却可能少一次攻击。这就是可观察、可验证的新增决策。

### 构筑与获取规则

- 战斗奖励先选择类别，再选择具体物品：
  - 指令档案：三张不同卡牌中选一张；
  - 城芯部件：两个模块中安装或升级一个；
  - 机体维护：恢复结构值。
- 模块奖励根据当前牌组行程值适度加权，但保留非最优候选。
- 工坊允许预览模块换位、升级后的效果和轨芯路线；确认前不得修改存档。
- 模块总负载上限为 6。强模块或高级模块占用更多负载，防止无脑堆满。
- 战斗界面必须显示本张牌结算后的轨芯落点预览，避免系统退化为心算负担。

## 4. 状态架构

```ts
type Scene =
  | "title"
  | "map"
  | "battle"
  | "reward"
  | "workshop"
  | "repair"
  | "victory"
  | "defeat";

interface SaveRoot {
  schemaVersion: 1;
  profile: ProfileState;
  run: RunState | null;
}

interface ProfileState {
  discoveredCardIds: string[];
  discoveredModuleIds: string[];
  settings: {
    musicVolume: number;
    sfxVolume: number;
    textSpeed: number;
  };
}

interface RunState {
  seed: string;
  scene: Scene;
  hull: number;
  maxHull: number;
  deck: CardInstance[];
  modules: InstalledModule[];
  moduleLoad: number;
  routeNodes: RouteNode[];
  routeLinks: RouteLink[];
  routeHistory: string[];
  currentLayer: number;
  pendingReward: RewardState | null;
  combat: CombatState | null;
}

interface CombatState {
  turn: number;
  actionPoints: number;
  armor: number;
  drawPile: string[];
  hand: string[];
  discardPile: string[];
  exhaustPile: string[];
  enemies: EnemyRuntime[];
  ringPosition: 0 | 1 | 2 | 3;
  ringDirection: 1 | -1;
  moduleRuntime: Record<string, ModuleRuntime>;
  phase: "player" | "enemy" | "resolving" | "won" | "lost";
}

interface InteractionState {
  selectedCardId: string | null;
  targetMode: "none" | "enemy" | "self" | "ring";
  validTargetIds: string[];
  previewRingPosition: number | null;
}
```

架构约束：

- 卡牌描述、战斗逻辑、奖励、升级预览和图鉴来自同一份卡牌数据。
- 模块说明、触发逻辑、负载、升级和界面提示来自同一份模块数据。
- DOM 只负责显示，不保存游戏真相。
- 伤害、轨芯移动、模块触发、路线合法性和奖励生成使用纯函数。
- 所有随机结果基于种子，失败局可以复现。
- 自动存档发生在节点选择、战斗开始、奖励确认和战斗结束后。
- 一次性阶段转换保存明确标记，防止首领变形、奖励或结算动画重复触发。
- 后续新增字段必须迁移旧存档，不能直接清空公开玩家进度。

## 5. 开发顺序

1. **无美术逻辑原型**  
   完成种子随机、卡牌解析、伤害、防护、抽弃牌、目标状态机和轨环移动的单元测试。

2. **单场多目标战斗**  
   使用纯色几何占位，验证选牌、目标、高亮、取消、无效目标和结束回合。

3. **轨环最小版本**  
   先接入 3 个模块，确认模块确实改变牌序；若测试者只看伤害数字、不看轨环，应先重做反馈，不扩内容。

4. **完整单局循环**  
   接通地图、两场战斗、奖励、工坊、首领、胜败结算和重新开始。

5. **构筑深度**  
   扩充至 18 张牌、6 个模块，增加奖励相关性、负载与升级预览。

6. **响应式界面**  
   建立桌面、普通横屏、手机横屏、极低高度横屏和竖屏提示五种布局状态。

7. **存档和兼容**  
   添加版本号、继续游戏、迁移函数及损坏存档回退。

8. **原创视听替换**  
   统一为高温沙晶、黑色陶甲、琥珀光路和巨大机械城剪影；不使用参考作品的卡框、图标、角色构图或文本风格。

9. **对抗式质检与公开部署**  
   完成逻辑、产物、真实浏览器和公网四层验证后才宣布可分享。

## 6. 可直接使用的提示词

### 开发主提示词

```text
从零实现原创浏览器牌组构筑 Roguelike《沙海机城》。

平台为电脑浏览器和手机横屏，最终部署为无需安装的公开静态网页。先冻结垂直切片：标题 → 真实分支地图 → 战斗 → 奖励 → 下一节点 → 两阶段首领 → 结算。

核心系统是“指令牌组 + 四站位城芯轨环”：每张牌包含费用、效果、目标和行程值；牌结算后轨芯移动，落点模块触发。模块必须影响牌序、目标或资源规则，不能只是常驻数值加成。

先定义数据结构、场景状态机、输入状态机和验收测试，再实现。规则不能散落在 DOM 事件中。卡牌和模块的效果、说明、升级、奖励与图鉴必须共享数据源。所有随机结果必须可由 seed 复现。
```

### 原创视觉提示词

```text
为《沙海机城》设计原创游戏视觉：横屏构图，巨型移动机械城穿越明亮沙海；材质为黑色陶甲、浅金砂晶、琥珀色能量线路和少量青白指示光。强调宏大尺度、清晰轮廓和机械结构，不使用航海元素、腐蚀金属、潮汐意象，也不模仿任何现有牌组构筑游戏的卡框、角色、图标、构图或文字。背景降低细节对比，战斗目标、意图和轨环必须优先可读。
```


## 7. 可验证验收标准

| 范围 | 通过标准 |
|---|---|
| 完整循环 | 新玩家无需说明文档，可从标题进入并完成地图、两场普通战斗、奖励、首领和结算 |
| 输入 | 选牌、目标高亮、确认、取消、无效目标和多目标选择全部符合状态机；无效点击不消耗资源 |
| 双构筑 | 至少三张牌可改变轨芯移动；至少三个模块影响牌序或目标；测试者能解释一次“为踩模块而改变出牌顺序”的决策 |
| 轨环一致性 | 100 个种子化回合中，预览落点、实际落点和模块触发日志完全一致 |
| 路线 | 所有可点击节点均存在真实连线；所有不可达节点不可点击；每个生成节点都有进入和继续路径 |
| 奖励 | 卡牌奖励三选一、模块奖励二选一；只能领取一次；预览和取消不会改变牌组或模块 |
| 首领 | 两阶段转换只触发一次；刷新页面继续后不重复回血、变形或发放奖励 |
| 响应式 | 1366×768、844×390、780×430、568×320 均能看到手牌、敌人意图、结构值、轨环和结束按钮 |
| 触控 | 主要点击目标约不小于 44×44 CSS 像素；不依赖 hover；横屏无阻塞式纵向滚动 |
| 断点 | 每个媒体查询边界前后 1px 均测试，无控件突然消失或叠压 |
| 存档 | 战斗、奖励和地图中刷新可继续；损坏存档安全回到标题；旧版本字段可迁移 |
| 原创性 | 无外部游戏名称、文本、角色、美术、音乐、代码或近似卡框；第三方素材均有明确商用许可记录 |
| 公开分享 | 游玩链接直接进入游戏；HTML、CSS、JS 和代表资源均返回 200；版本号与部署提交一致 |
| 发布回归 | 连续完成三局：胜利、失败、刷新继续各一局；控制台无未处理异常 |

第一版的真正完成条件不是“页面看起来像游戏”，而是陌生玩家能在手机横屏或电脑上完成一局，并能明确感受到：同一手牌因为轨芯位置不同，会产生不同的最优出牌顺序。
