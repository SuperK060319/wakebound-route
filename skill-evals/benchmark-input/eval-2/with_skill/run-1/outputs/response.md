# 《沙海机城》第一版产品规划

## 一、产品契约

《沙海机城》是一款原创浏览器牌组构筑 Roguelike。玩家扮演移动机城的调度师，在巨型沙丘带中维持城市运转、击退荒漠机械，并决定机城的构筑方向。

- 平台：电脑浏览器、手机横屏。
- 输入：点击卡牌，再点击敌人或己方机城；不依赖拖拽。
- 单局目标时长：第一版约 12～18 分钟。
- 核心循环：路线图 → 战斗 → 奖励 → 下一节点 → 首领 → 结算。
- 原创边界：名称、世界观、文案、数值、角色、敌人、图形和代码全部原创。
- 发布目标：静态网页公开部署，链接打开直接进入游戏首页。
- 下一里程碑：一名首次接触游戏的玩家无需教程讲解，即可独立打通一局。

一句话卖点：

> 你不只是在组牌，还要把每张牌产生的脉冲送入机城齿环；相同手牌因为出牌顺序不同，会启动完全不同的城市模块。

## 二、视觉与叙事方向

视觉命题为“夕照下仍在行走的巨型机械聚落”。

- 主色：曜黑、沙金、陶白、炽橙、暮紫。
- 材质：釉面陶甲、层压黑钢、琥珀玻璃、织物遮沙帘。
- 标志图形：同心齿环、脉冲刻度、折线式城区轮廓。
- 卡牌：像机城调度指令牌，不使用卷轴、古典羊皮纸式表达。
- 敌人：玻砂兽、废土钻机、热镜蜂群等荒漠机械生态。
- 状态动画：机城待机震动、模块启动、护甲展开、受击偏移、首领阶段切换。
- 文案语气：具体、克制，像工程命令和生存判断，避免广告腔。

## 三、第一版可玩垂直切片

### 1. 单局结构

路线共四层：

1. 第一层：从两个普通战斗节点中选择一个。
2. 第二层：在“整备台”和“齿序工坊”之间选择。
3. 第三层：第二场普通战斗，敌人组合受此前路线影响。
4. 第四层：首领“埋日掘城机”。

路线节点和路线连线分别存储。只能点击当前节点真实连线可达的下一节点，不能画出分支却允许任意跳转。

### 2. 首版内容量

- 1 座玩家机城。
- 52 点生命。
- 每回合抽 5 张牌，获得 3 点指令值。
- 10 张初始牌。
- 12 张完整卡牌数据。
- 5 种机城模块。
- 2 套普通遭遇。
- 1 个双阶段首领。
- 1 个整备节点。
- 1 个齿序工坊节点。
- 三选一卡牌奖励。
- 二选一模块奖励。
- 胜利、失败和重新开始流程。
- 当前进度、生命、货币、牌库与齿环状态始终可查。

### 3. 首版卡牌

初始牌组由前六种牌组成，共 10 张。

| 卡牌 | 费用 | 脉冲 | 效果 |
|---|---:|---:|---|
| 压进指令 ×3 | 1 | 1 | 对目标造成6点伤害 |
| 架盾指令 ×3 | 1 | 1 | 为机城获得5点护甲 |
| 校准击 | 1 | 0 | 造成4点伤害；若落点模块符合攻击条件，再造成3点伤害 |
| 惯性移位 | 0 | 2 | 抽1张牌 |
| 降载 | 1 | 0 | 获得7点护甲，弃1张牌 |
| 紧急停机 | 1 | 0 | 获得10点护甲，下回合少1点指令值 |
| 贯层钻 | 2 | 2 | 造成13点伤害 |
| 反压壁 | 1 | 2 | 获得8点护甲 |
| 连续脉冲 | 1 | 1 | 连续造成两次3点伤害，齿环只移动一次 |
| 回炉 | 1 | 0 | 弃1张牌，抽2张牌 |
| 错齿令 | 0 | 自选 | 将指针移动一格，不启动模块 |
| 过载开窗 | 1 | 2 | 获得1点指令值；本回合受到的下一次伤害增加2点 |

数值只是首轮测试基线，之后应根据通关率和实际决策记录调整。

## 四、原创双构筑系统：指令牌组 × 机城脉冲环

### 1. 系统规则

机城拥有一个三槽脉冲环，环上有一个当前指针。

每张牌都有一个醒目的脉冲数字：0、1 或 2。

一张牌成功结算时：

1. 先执行卡牌效果。
2. 指针顺时针移动对应格数。
3. 启动抵达位置的模块。
4. 模块读取本张牌的类别、目标和脉冲值，决定是否生效。

无效目标不会扣除费用、移动指针或启动模块。

### 2. 模块池

| 模块 | 一级效果 | 二级效果 |
|---|---|---|
| 冲桩器 | 由攻击牌抵达时，对该牌目标追加4点伤害 | 追加7点伤害 |
| 折砂闸 | 抵达时获得3点护甲；防御牌抵达则额外获得2点 | 基础护甲提高到5点 |
| 复写台 | 每回合首次由调度牌抵达时抽1张牌 | 同时使该牌费用返还1点 |
| 回弹轮 | 脉冲2的牌抵达时返还1点指令值，每回合一次 | 每回合可触发两次 |
| 换相键 | 抵达后，下一张牌的脉冲增加1，按0→1→2→0循环 | 玩家可以选择增加或减少1 |

### 3. 为什么它真的改变决策

假设当前指针位于0号槽：

- 1号槽是冲桩器；
- 2号槽是折砂闸；
- 0号槽是复写台；
- 敌人下回合将造成12点伤害；
- 手牌中有脉冲1攻击牌、脉冲1防御牌和脉冲2调度牌。

如果先攻击：

1. 攻击牌落到冲桩器，获得追加伤害。
2. 防御牌落到折砂闸，获得额外护甲。
3. 调度牌移动两格，回到复写台并抽牌。

如果先防御：

1. 防御牌落到冲桩器，模块不生效。
2. 攻击牌落到折砂闸，只获得普通护甲。
3. 同一组牌的总收益明显降低。

因此玩家每回合不只考虑“打哪张牌”，还要考虑：

- 牌的出牌顺序；
- 指针最终停在哪里；
- 是否为下一回合预留模块；
- 当前应该选择伤害、护甲还是循环效率；
- 奖励牌的脉冲值能否融入已有齿序；
- 新模块应该安装在哪个槽位。

### 4. 构筑获得与定向控制

- 战后奖励先选择“卡牌、模块、铸币”类别。
- 卡牌奖励展示三张不同候选，只能取一张。
- 模块奖励展示两个候选。
- 已拥有模块再次获得时，可以升级。
- 齿序工坊允许免费重排一次三个模块的位置。
- 奖励算法适度提高能补足现有构筑的模块权重。
- 所有模块效果、界面说明和战斗结算来自同一份模块数据。

这使第二构筑轴可以被看见、规划、定向获得和验证，而不是被动数值挂件。

## 五、战斗与交互契约

### 卡牌操作状态机

```text
空闲
├─ 点击攻击牌 → 选择敌方目标
├─ 点击防御牌 → 选择己方机城
└─ 点击无目标牌 → 显示确认落点并结算

选择目标
├─ 点击合法目标 → 结算一次并清除选择
├─ 点击非法目标 → 不扣费、不移环
├─ 再次点击已选卡牌 → 取消
├─ 点击空白区域 → 取消
└─ 按 Escape → 取消
```

移动端不强制显示跟随手指的细箭头，可改为明显的目标描边和连接光带，避免手指遮挡。

### 必须永久可见的信息

- 结束回合按钮。
- 当前指令值。
- 玩家生命与护甲。
- 敌人生命和下一行动意图。
- 当前路线层数。
- 齿环三个模块、当前指针及所选卡牌的预计落点。
- 抽牌堆、弃牌堆和牌库入口。

结束回合不能藏在手牌下面，也不能依靠玩家寻找快捷键。

## 六、状态架构

### 1. 场景状态机

```text
title
  → map
  → battle
  → reward
  → map
  → workshop / maintenance
  → map
  → boss
  → victory

battle / boss → defeat
```

存档直接保存场景名称和该场景必要的待处理数据，不通过 DOM 猜测玩家身处哪个页面。

### 2. 持久档案与单局状态分离

| 状态域 | 内容 |
|---|---|
| `profile` | 解锁项、已发现卡牌/模块、设置、音量、历史最高进度 |
| `run` | 种子、生命、货币、牌组、模块、指针、路线、节点历史、单局旗标 |
| `combat` | 手牌、抽牌堆、弃牌堆、指令值、护甲、敌人、意图、回合数、单回合触发记录 |
| `pending` | 奖励候选、工坊操作、胜负结算、尚未确认的选择 |
| `ui` | 当前选牌、目标模式、悬停卡牌、指针预览、弹窗状态 |
| `preferences` | 音量、动效强度、文字大小、是否显示详细战斗日志 |

`ui` 中纯视觉状态通常不写入存档；但奖励、工坊和战斗等会影响继续游戏的状态必须保存。

### 3. 单一数据源

建议拆分为：

```text
index.html
styles.css
src/
  content.js      卡牌、模块、敌人、节点数据
  state.js        初始状态与状态转换
  rules.js        伤害、护甲、齿环、奖励、路线纯函数
  scenes.js       场景流转
  render.js       界面渲染
  input.js        鼠标、触摸、键盘输入
  save.js         存档、版本与迁移
tests/
  rules.test.mjs
  run-smoke.test.mjs
```

关键纯函数：

- `resolveCard`
- `advancePulseRing`
- `triggerModule`
- `previewPulseLanding`
- `rollCardRewards`
- `rollModuleRewards`
- `makeRouteLinks`
- `routeNodeAvailable`
- `buildEnemyIntent`
- `migrateSave`
- `transitionScene`

存档至少包含 `schemaVersion`。旧存档缺少齿环、路线连线或奖励数组时，应补默认值或迁移，不能直接清空进度。

## 七、开发顺序

1. 建立卡牌、敌人、模块和状态数据结构，并为纯规则写测试。
2. 只做单场战斗：抽牌、费用、攻击、防御、敌人意图、结束回合、胜负。
3. 实现点击卡牌再点目标的完整状态机，包括取消和非法目标。
4. 加入三槽脉冲环、预计落点和5种模块，验证不同顺序产生不同结果。
5. 完成最小闭环：标题 → 地图 → 战斗 → 奖励 → 地图 → 首领 → 结算。
6. 加入真实路线连线、整备台、齿序工坊和保存/继续。
7. 统一战斗、卡牌奖励、模块奖励和工坊的组件语法。
8. 完成电脑与手机横屏布局，按可用高度压缩。
9. 增加原创图形、状态动画、帮助说明和基础音效。
10. 进行破坏性测试，逐项关闭问题。
11. 发布到固定公开地址，验证线上版本和缓存标识。

在完整闭环稳定前，不增加多角色、长期剧情、几十种敌人或复杂跨局养成。

## 八、首版明确不做

- 多角色与职业切换。
- 大型剧情档案。
- 云存档和账号系统。
- 多人排行。
- 超过12张的卡牌池扩张。
- 随机生成的大型地图。
- 完整配音。
- 纯数值式永久强化。
- 为手机竖屏重新设计完整战斗界面。

手机竖屏只显示明确的“请旋转设备”提示，并保证按钮仍可操作。

## 十、可验证验收标准

### 完整循环

- 新玩家可以从首页开始并完成整局。
- 每局必经两场普通战斗和一个首领。
- 战斗胜利后进入奖励，领取后回到地图。
- 首领胜利进入结算，失败进入失败页。
- 任意阶段刷新页面后可以继续当前场景。

### 战斗规则

- 合法攻击只作用于所选敌人。
- 防御牌只能以己方机城为目标。
- 非法目标不扣费用、不弃牌、不移动指针。
- 每张牌最多触发一次落点模块。
- 敌人死亡动画和首领阶段切换只播放一次。
- 结束回合连续点击不会让敌人行动两次。
- 牌面文字、预览和最终结算数值一致。

### 双构筑有效性

必须存在一条自动化用例证明：

- 使用同样的三张牌；
- 面对同样敌人和意图；
- 仅改变出牌顺序；
- 最终伤害、护甲或剩余指令值至少一项不同。

此外：

- 模块位置改变后，预计落点和实际触发同步改变。
- 模块升级前只显示预览，不提前消耗货币或修改状态。
- 模块奖励始终提供两个合法候选。
- 玩家在首领前至少有一次安装、升级或重排模块的机会。

### 路线

- 地图渲染使用的连线与节点可点击判断来自同一数据。
- 当前节点不能跳到没有连线的节点。
- 每个非终点节点至少有一条后续路径。
- 每个未来层节点至少存在一条进入路径。
- 相同种子生成相同路线、敌人与奖励序列。

### 响应式

通过以下实际视口：

- 1920×1080
- 1366×768
- 844×390
- 780×430
- 568×320
- 每个 CSS 断点前后1像素

通过条件：

- 无遮挡主要操作的纵向滚动。
- 结束回合始终可见。
- 五张起始手牌完整可读。
- 大手牌可横向滑动。
- 触控目标不小于约44×44像素。
- 浏览器工具栏压缩高度后，开始按钮仍可触达。
- 竖屏提示不会导致死路。

### 存档与兼容

- `profile` 和 `run` 使用不同存储键。
- 新开局不会删除永久设置与发现记录。
- 战斗、奖励、工坊、地图四种场景刷新后均可恢复。
- 旧版本缺少模块字段时能补默认值并继续。
- 损坏存档显示可理解的恢复提示，不出现白屏。

### 公开分享

- 提供的是直接游玩链接，不是仓库首页。
- 公开 HTML、CSS、JavaScript 和代表性资源均返回 HTTP 200。
- 页面资源带版本或内容哈希，更新后不会继续加载旧缓存。
- 线上页面显示的版本号与目标发布版本一致。
- 输出文件中不存在盘符路径、`file://` 或本地资源引用。
- 首次用户操作后才启动音频。
- 至少用一个真实手机浏览器完成首页进入、打一张牌、结束回合和刷新恢复测试。

## 十一、首版完成定义

第一版完成不是“页面已经画出来”，而是同时满足：

1. 一局可以从地图打到首领并正常结算。
2. 脉冲环让出牌顺序产生可验证的策略差异。
3. 路线选择真实约束后续节点。
4. 电脑和低高度手机横屏的主要操作均可见。
5. 存档、奖励和重复操作不存在明显状态漏洞。
6. 公开链接打开即玩，线上资源与目标版本一致。
7. 关键规则拥有自动化测试，视觉与触控拥有真实浏览器证据。

这套范围足以证明《沙海机城》的核心辨识度，同时仍然小到可以在第一轮开发中完成、测试和公开分享。
