# 一、产品合同

《沙海机城》是一款原创浏览器牌组构筑 Roguelike。

玩家扮演“巡构师”，驾驶一座行走于无边沙原上的机械聚落，依靠指令牌控制战斗，同时重排城内的环形动力轨道。每局约 15～25 分钟。

核心体验：

> 看清敌方意图，在“这张牌本身的效果”和“它会让城轨脉冲落到哪里”之间作取舍。

平台与交付：

- 电脑浏览器、平板横屏、手机横屏。
- 鼠标和触控均使用“点牌—点目标”，不依赖拖拽。
- 静态 HTML/CSS/JavaScript 即可部署。
- 发布时提供直接游玩链接，而不是仓库页。
- 保留可选的单文件 HTML，方便转发和离线演示。

原创边界：

- 世界观、名称、角色、敌人、卡牌、数值、UI、图像、声音和代码全部原创。
- 视觉采用“陶瓷装甲、青金釉面、黑曜玻璃、白炽能源、热浪折射”的沙原机械文明。
- 不复用参考作品的角色轮廓、卡框、图标、地图造型、文本、音效或具体数值。
- 不采用题目明确排除的主题材料和术语。

# 二、第一版可玩垂直切片

## 1. 完整单局循环

```text
标题页
→ 路线地图
→ 普通战斗
→ 奖励三选一
→ 路线地图
→ 整备站或城区工坊
→ 第二场普通战斗
→ 奖励
→ 首领战
→ 胜利/失败结算
→ 返回标题页
```

地图采用四层节点，边线真实限制下一步，而不是所有节点都能互相跳转。

第一版内容范围：

- 1 名可玩巡构师。
- 10 张初始牌。
- 12 种卡牌定义。
- 6 种可安装城区模块。
- 2 套普通遭遇。
- 1 个双单位遭遇，用来验证目标选择。
- 1 个两阶段首领。
- 1 个整备节点。
- 1 个城区工坊。
- 1 张可复现的种子路线。
- 胜利、失败和重新开始。
- 单局自动保存与继续游戏。

暂不加入：

- 跨局等级成长。
- 大型故事章节和多结局。
- 数十种敌人。
- 完整商店经济。
- 多角色。
- 联网排行。
- 复杂配音。
- 大量随机事件。

## 2. 基础战斗规则

- 每回合获得 3 点“调度”。
- 抽取 5 张指令牌。
- 敌方下一步行动永久可见。
- 攻势牌：点牌后点敌人。
- 架势牌：点牌后点自己的机城。
- 调整牌：满足条件时直接结算，涉及选择时弹出明确目标。
- 点击已选牌、空白区域或按 Escape 取消。
- 无效目标不扣调度、不弃牌。
- “结束调度”按钮固定在操作层，任何支持的横屏尺寸都不可被手牌遮挡。
- 护幕默认在己方下一回合开始时清除。
- 每张牌结算后，处理城轨脉冲，再进入下一次输入。

## 3. 第一版卡牌

初始牌组为 10 张：3 张“切割令”、3 张“立幕令”，其余四种各 1 张。

| 卡牌 | 消耗 | 基础效果 | 轨距 |
|---|---:|---|---:|
| 切割令 | 1 | 对一个敌人造成 6 点伤害 | 1 |
| 立幕令 | 1 | 获得 5 点护幕 | 1 |
| 短程校准 | 0 | 抽 1 张牌，本牌不推进脉冲 | 0 |
| 偏转令 | 1 | 获得 4 点护幕，改变脉冲方向 | 1 |
| 过站冲击 | 1 | 造成 4 点伤害 | 2 |
| 负载转移 | 1 | 本回合下一个城区模块可以额外启动一次 | 1 |
| 三段迭进 | 1 | 造成 5 点伤害；若落点模块尚未启动，再造成 3 点 | 3 |
| 停机护持 | 1 | 获得 9 点护幕 | 0 |
| 联供脉冲 | 1 | 下一次城区效果增强 50% | 2 |
| 清障轰击 | 2 | 造成 13 点伤害 | 1 |
| 余量回收 | 0 | 若脉冲落在本回合已启动的模块，抽 1 弃 1 | 1 |
| 强制换轨 | 1 | 将脉冲移到相邻模块，不触发当前落点 | 可选 ±1 |

升级不只增加数值。例如：

- “过站冲击+”允许结算前选择轨距 1 或 2。
- “停机护持+”在不推进脉冲的同时重置一个模块。
- “强制换轨+”可以移动至相隔两个位置的模块。

这样，升级会改变路线计算，而不只是让数字变大。

## 4. 敌人与首领

普通遭遇一：“掘沙足群”

- 两个低生命单位。
- 一个准备连续攻击，另一个准备削弱护幕。
- 用于验证多目标选择和敌方意图。

普通遭遇二：“镜甲收割机”

- 单个中等生命敌人。
- 每隔一回合展开镜甲，使下一次城区直接伤害降低。
- 迫使玩家选择先用基础牌破除防护，还是调整脉冲落点。

首领：“吞日钻塔”

第一阶段：

- 高伤攻击、蓄能、封锁城区三种意图轮换。
- 被封锁的轨位下一回合无法启动，但脉冲仍可经过或停留。

第二阶段：

- 生命降至 50% 时进入一次性转换状态。
- 强制反转脉冲方向。
- 每两回合封锁两个不相邻轨位。
- 转换动画只播放一次，状态稳定后不得因重新渲染重复播放。

首领直接检验第二构筑轴是否真正影响战斗，而不是战斗外的被动加成。

# 三、原创双构筑系统：牌组 × 城轨脉冲盘

## 1. 可见载体

战场中央下方显示一个六轨位环形“城轨脉冲盘”。

- 六个轨位可安装城区模块。
- 一个发光脉冲停留在其中一格。
- 每张牌拥有 0～3 的“轨距”。
- 牌结算后，脉冲按当前方向精确移动对应格数。
- 最终落点上的城区模块被启动。
- 脉冲位置和方向跨回合保留。
- 同一模块默认每回合只能启动一次。

玩家不只是在构筑牌库，也在构筑一座会被牌序驱动的机械城市。

## 2. 为什么它真的改变决策

假设敌人下回合攻击 10 点，当前脉冲位于 0：

- 轨位 1 是“裂光炮街”。
- 轨位 2 是“折幕井”。
- 玩家手里有轨距 1 的攻击牌和轨距 2 的防御牌。

玩家会面临真实取舍：

- 先打轨距 1 的牌，会获得牌伤害并启动炮街，但脉冲停在 1。
- 先打轨距 2 的防御牌，会直接落在折幕井，获得牌护幕和城区护幕。
- 如果之后再打轨距 1，脉冲会从 2 移到 3，而不是回到炮街。
- 若牌序反过来，最终伤害、护幕、落点和下一回合起始状态都会不同。

因此，一张基础效果稍弱但轨距合适的牌，可能比高伤牌更值得拿；城区安装位置也会反过来改变玩家想要的牌组轨距分布。

## 3. 第一版六种城区模块

| 城区模块 | 启动效果 | 构筑作用 |
|---|---|---|
| 裂光炮街 | 对攻击意图最高的敌人造成 4 点伤害；由攻势牌启动时再加 2 | 鼓励攻势牌与精确落点 |
| 折幕井 | 获得 4 点护幕；由架势牌启动时额外获得 2 | 防御牌与城区协同 |
| 调度环 | 若本次移动轨距正好为 2，返还 1 点调度，每回合一次 | 让轨距 2 成为牌组方向 |
| 变轨塔 | 选择脉冲之后顺时针或逆时针移动 | 改变后续牌序规划 |
| 回收庭 | 抽 1 张牌，再弃 1 张牌 | 调整手牌并寻找特定轨距 |
| 锁距台 | 下一张牌结算时，其轨距可选择 +1 或 -1，最低为 0 | 提供精确导航能力 |

## 4. 安装、升级和组合

- 开局拥有三个基础模块和三个空轨位。
- 战斗奖励中的“城区方案”一次展示两个模块，选择一个安装或升级。
- 工坊允许付费移动两个模块的位置。
- 整备站允许修复生命，或免费旋转一次城区布局，二选一。
- 安装和重排必须先显示预览，确认前不扣资源、不改状态。

邻接组合示例：

- “调度环”与“裂光炮街”相邻时，首次启动炮街后可让下一张轨距 1 的牌视为轨距 2。
- “折幕井”与“锁距台”相邻时，通过锁距修正落入折幕井，会保留 3 点护幕到下一回合。
- “回收庭”与“变轨塔”相邻时，改变方向后可查看抽牌堆顶两张并选择一张置顶。

第一版只实现前两组组合，第三组作为后续扩展，防止垂直切片膨胀。

## 5. 定向获得，避免纯随机

卡牌奖励：

- 展示 3 张不同卡牌。
- 牌池会考虑当前牌组轨距分布。
- 如果玩家安装了依赖轨距 2 的调度环，轨距 2 卡牌权重提高，但不是必出。

城区奖励：

- 展示 2 个不同模块。
- 优先提供现有模块的组合伙伴或空缺功能。
- 工坊提供确定性的移动和升级渠道。

玩家能够“追一个构筑”，而不是等待唯一随机掉落。

# 六、开发顺序

## 阶段 0：冻结规则

产物：

- 产品合同。
- 12 张牌的数据表。
- 6 个城区模块。
- 两场普通战斗和一个首领。
- 固定的交互状态机。
- 明确“不做清单”。

通过门槛：没有尚未决定的核心输入方式、资源名称或战斗顺序。

## 阶段 1：纯状态与命令行逻辑

先实现：

- 抽牌、弃牌、调度。
- 伤害、护幕、敌方意图。
- 轨距移动、模块启动、每回合启动限制。
- 路线边。
- 奖励选择。
- 胜负判定。
- 序列化和迁移。

通过门槛：不依赖浏览器画面即可跑通一局确定性测试。

## 阶段 2：占位视觉的完整循环

使用简单几何图形完成：

```text
地图 → 战斗 → 奖励 → 地图 → 首领 → 结算
```

通过门槛：第一次接触游戏的人无需开发者指导，可以完成一局。

## 阶段 3：验证双构筑

加入：

- 六轨位脉冲盘。
- 城区安装与重排。
- 卡牌轨距。
- 两组邻接组合。
- 首领封锁轨位。

通过门槛：至少有三组自动化场景证明改变牌序、城区位置或脉冲方向会改变战斗结果。

## 阶段 4：信息和防误触

加入：

- 卡牌与城区提示。
- 轨距移动预览。
- 有效目标高亮。
- 工坊移动前后预览。
- 重新开始二次确认。
- 图鉴或规则帮助页。

通过门槛：确认前状态不发生改变；无效目标不消耗任何资源。

## 阶段 5：响应式与触控

完成桌面、平板和低高度手机横屏布局。

通过门槛：所有目标视口能看到结束调度、生命、调度点、敌人意图和手牌主信息。

## 阶段 6：原创视觉与音频

视觉：

- 沙原热浪。
- 陶瓷装甲机械城。
- 黑曜玻璃敌人。
- 脉冲沿轨道移动的明确动画。
- 攻击、护幕、命中、封锁、首领转换均有不同反馈。

音频：

- 标题、地图、普通战斗、首领、奖励分别路由。
- 浏览器首次点击后才启动声音。
- 所有素材记录许可证、来源、署名要求和文件大小。

通过门槛：关闭动画后仍能靠文字和状态读懂规则；关闭声音不影响操作。

## 阶段 7：保存与公开发布

- 使用版本化存档。
- 自动保存地图、战斗和待领奖励。
- CSS/JS 加版本标识。
- 部署到稳定公开地址。
- 生成可选单文件 HTML。

通过门槛：刷新后不会重复奖励、重置首领阶段或改变随机结果。

## 阶段 8：对抗性测试

主动寻找 10 个非重复问题，优先检查：

- 连点卡牌。
- 动画未完时结束回合。
- 无效目标。
- 被击败敌人再次被选中。
- 首领转换重复播放。
- 已启动模块重复启动。
- 路线边与可点击节点不一致。

每个问题独立修复、独立测试，不用一个大改包住全部问题。

# 八、可验证验收标准

| 类别 | 通过条件 |
|---|---|
| 完整循环 | 新玩家能从标题页进入，并在 25 分钟内完成地图、两场普通战斗、两次奖励、首领和结算 |
| 战斗输入 | 点牌—点目标可完成攻击和防御；取消、无效目标和已击败目标均不扣资源 |
| 主操作 | 所有目标横屏尺寸中，“结束调度”始终可见且可点击 |
| 敌方意图 | 每个存活敌人都显示下一步动作；结算后意图只推进一次 |
| 双构筑 | 至少三个确定性测试证明不同牌序、模块位置或脉冲方向产生不同伤害、护幕或资源结果 |
| 轨距正确性 | 0、1、2、3 轨距、反向移动和 ±1 修正均有边界测试 |
| 模块限制 | 同一模块每回合只启动一次；额外启动必须由明确效果授权 |
| 首领封锁 | 被封锁轨位不启动，但脉冲位置仍按规则移动；UI与逻辑一致 |
| 奖励构筑 | 卡牌展示 3 个不同候选；城区展示 2 个不同候选；只能领取一次 |
| 定向构筑 | 安装调度环后，大量固定种子采样中轨距 2 卡牌出现率有可测提升，但不是 100% |
| 真实路线 | 可点击节点集合必须严格等于上一节点边线指向集合；每条路线可到达首领 |
| 预览确认 | 工坊重排、升级和重新开始在确认前不改状态、不扣资源 |
| 存档恢复 | 在地图、战斗、奖励和首领第二阶段刷新页面，均恢复到正确场景且不重复结算 |
| 存档迁移 | 缺少脉冲方向、城区回合状态或路线边的旧存档可补默认值并继续运行 |
| 一次性动画 | 首领阶段转换和敌人被击败动画只播放一次，重新渲染不重播 |
| 响应式 | 1366×768、844×390、780×430、568×320 以及断点前后 1px 无主操作遮挡 |
| 触控 | 主要按钮约 44×44 CSS 像素；手牌横滑不产生遮挡卡面的纵向滚动条 |
| 竖屏处理 | 手机竖屏显示旋转提示；不会呈现不可操作的压缩战斗界面 |
| 原创性 | 资产清单中每个图像、字体、音乐和音效都有原创或合法授权记录，无外部本地路径 |
| 静态产物 | JavaScript 语法、资源引用、选择器和单文件构建检查通过 |
| 公开交付 | 直接游玩链接返回 200；HTML/CSS/JS 版本一致；代表资源可加载；部署对应目标提交 |
| 缓存 | 发布新版本后公共 HTML 引用新的资源版本标识，旧缓存不会继续加载旧规则 |
| 首次音频 | 未经用户手势不自动播放；首次点击后才能启动音乐 |
| 对抗性测试 | 至少 10 个非重复风险分别拥有测试名称、修复和通过证据 |

第一版的停止条件不是“页面已经很好看”，而是：完整单局可以公开打开、双构筑确实改变牌序和选牌、目标横屏尺寸可操作、存档可恢复、所有上述关键验收项拥有可重复证据。
