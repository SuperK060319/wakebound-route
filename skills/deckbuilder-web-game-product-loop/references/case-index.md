# Case Index for Fast Cold Starts

Use this short index for planning-only tasks. Read the full `case-library.md` before implementing or auditing the corresponding behavior.

| ID | Trigger signal | Root lesson |
|---|---|---|
| C01 | “结束回合在哪里” | Primary actions must be discoverable at every target viewport. |
| C02 | “一屏装下” + “看不到进度” | Separate competing surfaces into scenes; keep a persistent summary. |
| C03 | Drag/click interaction correction | Model input as a selected-card/target/cancel state machine and validate with multiple targets. |
| C04 | Symbols feel lifeless | Actors need stateful animation; one-shot transitions need persistent markers. |
| C05 | Local path/repository is not playable | Direct public entry, cache identity, portable build, and source repository are different artifacts. |
| C06 | Defense feels disconnected | Symmetric attack/defense concepts should share targeting grammar when it improves clarity. |
| C07 | Need music candidates | Screen license objectively, let the user choose subjective fit, then route by scene. |
| C08 | “Not only cards; upgrade the ship” | A signature second construction axis must change decisions and be intentionally obtainable. |
| C09 | Rewards look like three games | Equal hierarchy means shared geometry, information zones, and interaction—not only shared colors. |
| C10 | Upgrade/remove/restart misclick | Preview costly changes; confirm irreversible actions; preview must not mutate state. |
| C11 | “What does this status do?” | Every mechanic needs in-context help and a full reference derived from the same data. |
| C12 | Reference settings screen | Import useful behavior, not irrelevant decorative options; test real state changes. |
| C13 | Cards clipped on narrow phones | Overflow is two-dimensional; test transform lift, browser bars, width and height together. |
| C14 | Branching map is decorative | Rendered route edges and click eligibility must share one seeded graph. |
| C15 | Real WeChat screenshots fail | Remote-device evidence outranks local confidence; isolate the rendering layer that failed. |
| C16 | Prototype lacks depth | Expand around archetypes and encounter questions, not raw content totals. |
| C17 | 780px works, 812px fails | Test both sides of every breakpoint; preserve 44px touch targets and critical HUD state. |
| C18 | Ten audit problems | Give every issue an independent state contract, regression group, and public-release proof. |

Fast routing examples:

- New original game: C02 + C03 + C05 + C08 + C14 + C17.
- Mobile overlap: C01 + C13 + C15 + C17.
- Reward/build depth: C08 + C09 + C10 + C11.
- Story/save: C10 + C14 + C16 + C18.
- Public release: C05 + C15 + C17 + C18.
